## But first&#8230;The Books? The Blu-rays?

Actually the (non sheet-music) books and all the Blu-rays/DVDs etc have been done. As part of downsizing from a big house to smaller apartment, perhaps 75% were donated to charities.

It was fairly easy to catalogue the remaining items here at the new place. I can recommend the free [LibraryThing ](https://www.librarything.com/)cloud-based system for this. 

After you&#8217;ve set up an account, you can install a phone/tablet app. And/or use a browser.

Then just enter your books and it tries to retrieve the full data record for it (Title, Author, Reviews etc etc). In order of ease-of-use, I did this:

- Scan the book&#8217;s barcode with the phone&#8217;s camera (magic!)
- If barcode not available or found: manually type in the ISBN 
- If ISBN fails, search by Title/Author etc 
- If all that fails, then manually enter a new entry. You can even photograph the cover and upload it.

Metadata (tags) are automatically added for the &#8216;found&#8217; ones. Or you can add your own. You can add and search etc from the browser or your mobile device.

It also supports other things Blu-rays, X-Box Games and  you can add your own media or item types.

## The Project Itself: Step One

The first challenge is to work out what sort of cataloguing system can I use for the Photos, Video Recordings etc. 

Something similar to LibraryThing (LT)?

LibraryThing itself?  It may not be suitable or flexible enough. It is very &#8216;book&#8217; based; officially everything is a &#8216;book&#8217;. Plus it&#8217;s a closed, proprietary system AFAIK (of course you can easily export your data and in multiple formats) .   If they vanish or start charging (or charging more, see below), then grrr.  

A great note-taking system called *Evernote* did just this; free, hooked you, then brought in fees to do what used to be free!

Another, lesser, consideration is that the free version of LT has item limits, which I&#8217;ll think I&#8217;ll hit. 

So I&#8217;m looking for a more open Catalogue system, where I can:

- Store the database on, say, my own Dropbox folder
- Use it from a phone app, tablet app and browser

Besides the Sheet Music there&#8217;s no need to use an Internet &#8216;look up&#8217; items as I catalogue them. I&#8217;ll be manually entering what each item is. 

Which means I have to think about *what *I record for each thing. Early thoughts are:

- ID (unique;  &#8216;slides001&#8217;, book&#8217;s-title, tape&#8217;s-name)
- Media Type  
- Label (what else is written on it)
- Date  (not as simple as you think, some photo-packs are just a Year, or &#8211; worse &#8211; Year??)
- Contents (say, a list of songs for the Song Books, trips/events on a given video tape)
- Keywords (family, work, person&#8217;s name)
- Comments
- Important/Interesting (boolean; Y or N)
- Storage location (photobox 1980s-02)

I&#8217;d like to be able to have all of these as database indexes, or at least be able to search/sort on them.

It would have to be able to **import **existing, simple Excel catalogues (or via CSV) and even &#8211; ideally &#8211; LibraryThing exports, enabling me to have everything in one place.
