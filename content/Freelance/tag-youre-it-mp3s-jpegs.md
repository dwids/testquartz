What do your digital photographs and your MP3 music files have in common? More than you might think, for at the heart of our modern digital lifestyle are two things: data compression and tags.

**Tags** are little pieces of data that are &#8220;inside&#8221; the JPEG or MP3. They are part of the file itself and go with it when you move it to your player, email it to a friend or copy it to your notebook. If they were stored &#8220;outside&#8221; of the file, this data wouldn&#8217;t automatically be transferred with the file.

In the case of MP3s, the tags contain information on the artist, song title, album title, genre (rock, folk, blues) and duration. So even a file with a bland name like music01.mp3 can actually have meaningful tag data when loaded up into iTunes or Winamp. These players read the tags and show that information &#8211; for instance, *Hold My Hand* by *Johnny and the Moondogs.*

There are a number of useful programs to display and change the tags, including some basic functions in iTunes and Winamp.

An excellent one is the free and open source MP3 Book Helper.

Not only can it change the tags themselves but it can use the tag information to rename the file. A common layout is artist-title. So music01.mp3 would be renamed *Johnny and The Moondogs &#8211; Hold My Hand.mp3.*

The JPEGs have similar data more relevant to digital photographs; date and time taken, camera model, lens and exposure settings, flash information, and a whole lot more. Again, useful programs such as the free Picasa2 can use the tags to show more friendly views of your pictures, for example all the shots taken on a particular date. Sure beats file names like DSC-0684.JPG.

These tags are excellent examples of a broader technical idea called metadata. Wikipedia describes **metadata **as &#8220;data about data&#8221;.

A common example might be a library catalogue card, which contains data about the contents and location of a book: it is data about the data in the book referred to by the card.

These metadata tags are also at the heart of the World Wide Web.

They are &#8220;hidden&#8221; elements of webpages, not normally visible to end users in the browsers.

But they are quite visible to search engines. Indeed that&#8217;s where the search engines look first when they find your webpage and try to work out what that page is all about.

If you are browsing a webpage, say the *Museum Victoria* site, you can actually see these tags. In your browser there should be a menu option such as &#8220;View Source&#8221;. In Firefox it is under the View menu and called &#8220;Page Source&#8221;.

Doing this on the museum&#8217;s page will show metadata like this:
![](3_image.png)(new image, original lost)
Then along comes a spider (part of a search engine that finds webpage information) and it quickly finds these keywords. It then has a good, high-level summary of what this page &#8211; and site &#8211; are all about.

The spider can add these words to the huge index of the search engine and associate them with that website.

Later, we can go to the search engine&#8217;s site and type in one or more of these words (*Museums Victoria*) and, ta da, it returns the link for that website.
